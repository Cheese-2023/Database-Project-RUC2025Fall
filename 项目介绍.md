# 县域风险预警与可视化决策系统 - 项目详细文档

## 一、项目背景与目标

### 1.1 项目背景

县域经济作为中国国民经济的基础单元，其健康发展直接关系到国家整体经济稳定和社会和谐。随着经济社会的快速发展，县域面临着经济、社会、环境等多方面的风险挑战。如何及时识别、评估和预警这些风险，为决策者提供科学依据，成为当前亟待解决的问题。

传统的风险监测方式主要依赖人工统计和简单的数据分析，存在以下问题：
- **数据分散**：县域相关数据分散在多个部门和系统中，难以统一管理
- **分析滞后**：缺乏实时、动态的风险评估机制
- **决策支持不足**：缺乏可视化的数据展示和智能化的分析工具
- **预警机制不完善**：无法及时识别和预警潜在风险

### 1.2 项目目标

本项目旨在构建一个**县域风险预警与可视化决策系统**，实现以下目标：

1. **数据整合**：整合县域经济、社会、环境、人口等多维度数据，建立统一的数据管理平台
2. **风险评估**：建立科学的县域风险评估模型，实现多维度、多指标的综合风险评价
3. **实时预警**：构建自动化的风险预警机制，及时发现高风险县域
4. **可视化展示**：提供直观的数据可视化界面，支持多维度数据分析和趋势展示
5. **智能辅助**：集成AI助手功能，为决策者提供智能化的数据分析和建议
6. **成果展示**：专门展示832个国家级贫困县脱贫摘帽后的经济发展成果

### 1.3 应用场景

- **政府部门**：用于县域风险监测、政策制定和资源配置决策
- **金融机构**：用于信贷风险评估和投资决策
- **研究机构**：用于县域发展研究和政策分析
- **企业组织**：用于市场分析和投资选址决策

## 三、数据库设计

### 3.1 数据库概述

系统采用MySQL 8.0作为主数据库，数据库名称为`county_risk_warning_system`，字符集为`utf8mb4`，支持完整的Unicode字符集。

数据库共包含**23张表**，涵盖以下数据域：

1. **基础信息域**（2张表）
   - 县域基础信息
   - 行政区划结构

2. **统计数据域**（10张表）
   - 人口统计
   - 就业统计
   - 经济总量
   - 财政金融
   - 农业统计
   - 工业统计
   - 基础设施
   - 投资消费
   - 教育卫生
   - 环境文化

3. **风险评估域**（3张表）
   - 风险指标配置
   - 综合风险评估
   - 风险维度得分

4. **预警管理域**（3张表）
   - 用户表
   - 预警规则
   - 预警记录

5. **系统管理域**（5张表）
   - 数据质量检查
   - 系统配置
   - 操作日志等

### 3.3 数据关系设计

系统采用**星型模型**设计，以`county_basic`表为中心，其他统计表通过`county_code`和`year`关联：

```
county_basic (中心表)
    ├── population_statistics (人口统计)
    ├── economic_aggregate (经济总量)
    ├── fiscal_finance (财政金融)
    ├── employment_statistics (就业统计)
    ├── ... (其他统计表)
    └── comprehensive_risk_assessment (风险评估结果)
```

所有统计表都通过`county_code`和`year`建立唯一索引，确保数据的唯一性和查询效率。

## 五、风险评估算法

### 5.1 评估模型架构

系统采用**多维度加权综合评估模型**，将县域风险分解为5个维度：

1. **经济风险**：GDP增长率、人均GDP、财政自给率、政府债务率等
2. **社会风险**：人口流失率、就业率、城乡收入差距等
3. **环境风险**：空气质量、污染排放强度等
4. **治理风险**：教育投入、公共服务水平等
5. **发展风险**：投资效率、消费率、创新能力等

### 5.2 计算流程

```
数据输入
    ↓
指标数据提取（从各统计表）
    ↓
单项指标得分计算
    ↓
维度风险得分计算（加权求和）
    ↓
综合风险得分计算（维度加权平均）
    ↓
风险等级判定
    ↓
结果存储
```

### 5.3 指标得分计算

对于每个风险指标，系统根据指标值和阈值计算得分：

**对于"小于阈值风险高"的指标（LT类型）**：
- 如果 指标值 < 高风险阈值 → 得分 = 100
- 如果 指标值 < 中风险阈值 → 得分 = 80
- 如果 指标值 < 低风险阈值 → 得分 = 60
- 否则 → 得分 = 20

**对于"大于阈值风险高"的指标（GT类型）**：
- 如果 指标值 > 高风险阈值 → 得分 = 100
- 如果 指标值 > 中风险阈值 → 得分 = 80
- 如果 指标值 > 低风险阈值 → 得分 = 60
- 否则 → 得分 = 20

### 5.4 综合得分计算

**维度风险得分** = Σ (指标得分 × 指标权重)

**综合风险得分** = (经济风险得分 + 社会风险得分 + 环境风险得分 + 治理风险得分 + 发展风险得分) / 5

### 5.5 风险等级判定

根据综合风险得分，系统将县域划分为5个风险等级：

| 风险等级 | 得分范围 | 说明 |
|---------|---------|------|
| 低风险 | < 22.5 | 风险较低，发展稳定 |
| 中低风险 | 22.5 - 23.5 | 风险较低，需关注 |
| 中风险 | 23.5 - 24.2 | 风险中等，需加强监测 |
| 中高风险 | 24.2 - 25.0 | 风险较高，需重点关注 |
| 高风险 | ≥ 25.0 | 风险很高，需立即采取措施 |

### 5.6 模型可配置性

系统支持动态调整风险评估模型：
- **指标权重**：可调整各指标在维度内的权重
- **风险阈值**：可调整高/中/低风险的判定阈值
- **维度权重**：可调整各维度在综合得分中的权重（当前为平均权重）

## 六、用户权限系统

### 6.1 角色设计

系统设计了5种用户角色，实现细粒度的权限控制：

1. **ADMIN（系统管理员）**
   - 全部权限
   - 可访问所有功能模块
   - 可调整所有配置

2. **USER（普通用户）**
   - 基础查看权限
   - 仅可访问风险监控大屏

3. **VIP（VIP用户）**
   - USER权限 + AI助手
   - 可访问风险监控大屏和AI助手

4. **RISK_ANALYST（风险分析员）**
   - VIP权限 + 数据增删改 + 风险参数调整 + 全部页面
   - 可访问所有功能模块
   - 可调整风险计算参数
   - 可执行SQL操作

5. **DATA_MAINTAINER（数据维护员）**
   - VIP权限 + 数据增删改 + 全部页面
   - 可访问所有功能模块
   - 可执行SQL操作
   - 不可调整风险参数
